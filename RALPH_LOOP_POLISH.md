# JayGame 폴리싱 Ralph Loop 프롬프트

## 목표
운빨존많겜 수준의 **프리미엄 모바일 게임 UI/UX** 달성.
매 반복마다 아래 체크리스트에서 ❌인 항목 하나를 골라 구현하고, 완료 시 ❌→✅로 변경 후 커밋.

## 핵심 원칙
1. **빌드 깨지면 안 됨** — 매 작업 후 `./gradlew assembleDebug` 성공 확인
2. **기존 구조 유지** — Compose Canvas 렌더링, BattleBridge StateFlow, 엔진 레이어 구조 절대 변경 금지
3. **그리드 뷰 절대 금지** — 유닛은 자유 이동만 (Grid 내부 관리용일 뿐)
4. **GC 최소화** — Canvas 내에서 Color/Paint 객체 프레임마다 생성 금지, top-level pre-allocate
5. **한 번에 하나씩** — 한 반복에 체크리스트 항목 1~2개만 작업. 작은 단위로 커밋
6. **한국어 UI** — 모든 유저 대면 텍스트는 한국어

### 배틀필드 좌표 맵 (현재 → 목표)

**현재 (직사각형, 문제 있음):**
```
1280×720 세계좌표
Grid: (200,140)~(1080,580) = 880×440, cell 140×88
Path: (130,70)~(1150,650) = 1020×580, margin 70px
Waypoint: (165,105)→(1115,105)→(1115,615)→(165,615)
문제: 상변 950px, 측변 510px → 속도 불균일
```

**목표 (정사각형):**
```
1280×720 세계좌표
Grid: (400,120)~(880,600) = 480×480, cell 96×96 (5×5), TOTAL=25
Path: (340,60)~(940,660) = 600×600, margin 60px
Waypoint: 정사각형 + 코너 보간 8~12점
결과: 4변 동일 길이 → 균일 이동
```

**좌표 동기화 필수 파일:**
- `Grid.kt`: ORIGIN_X/Y, CELL_W/H, COLS/ROWS
- `BattleEngine.kt`: enemyPath waypoints
- `MonsterPathOverlay.kt`: PATH_LEFT/TOP/RIGHT/BOTTOM, GRID_LEFT/TOP/RIGHT/BOTTOM
- `BattleField.kt`: GRID_NORM_X/Y/W/H
- `GameUnit.kt`: wander 범위 clamp

**절벽 렌더링 (Canvas만으로):**
```kotlin
// 1. 상단면 (그라운드)
drawRoundRect(groundColor, topLeft, Size(480,480), cornerRadius=12)

// 2. 측면 (절벽) — drawPath로 사다리꼴
val cliffPath = Path().apply {
    moveTo(gridLeft, gridBottom)           // 좌하단
    lineTo(gridRight, gridBottom)          // 우하단
    lineTo(gridRight + 8, gridBottom + 25) // 우하단 아래 (약간 넓어짐)
    lineTo(gridLeft - 8, gridBottom + 25)  // 좌하단 아래
    close()
}
drawPath(cliffPath, cliffColor)  // 어두운 갈색

// 3. 그림자
drawOval(shadowColor, below grid, spread)
```

// 무한 반복 글로우
val glow by rememberInfiniteTransition().animateFloat(

)

// 스프링 바운스 등장
val scale by animateFloatAsState(
    targetValue = if (visible) 1f else 0f,

val particles = remember { Array(MAX) { Particle() } }
