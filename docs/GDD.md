## 1. 게임 개요 (Game Overview)

### 1.1 기본 정보

| 항목 | 내용 |
|------|------|
| **게임명** | JayGame (제이게임) |
| **장르** | 전략 랜덤 디펜스 (Strategic Random Defense) |
| **플랫폼** | Android (모바일) |
| **엔진** | 자체 제작 (C++ NDK + OpenGL ES 3.0) |
| **타겟 유저** | 15~35세, 모바일 캐주얼/미드코어 게이머 |
| **세션 길이** | PvP: 3~7분 / PvE: 15~30분 |
| **비즈니스 모델** | F2P + 광고 + 시즌패스 + 인앱 구매 |
| **minSdk** | API 26 (Android 8.0) |
| **논리 해상도** | 1280x720 |

### 1.2 핵심 컨셉

> **"운에 기대되 실력으로 이긴다"**

JayGame은 랜덤 소환의 긴장감과 전략적 합성 시스템의 깊이를 결합한 모바일 랜덤 디펜스 게임이다. 플레이어는 5개의 유닛 덱을 구성하고, 전투 중 랜덤으로 소환되는 유닛을 합성하여 강화하며, 밀려오는 적들을 방어한다.

### 1.3 차별점

| 비교 항목 | Random Dice | SC/메이플 랜덤 디펜스 | **JayGame** |
|-----------|------------|---------------------|-------------|
| 그리드 | 5x3 (15슬롯) | 가변 | **5x4 (20슬롯)** — 전략 공간 확대 |
| 합성 결과 | 랜덤 변경 | 조합 테이블 | **선택지 제공** (유지 OR 랜덤) |
| 세션 길이 | 3~7분 | 30~60분 | **PvP 5분 / PvE 20분** |
| 유닛 역할 | 주사위 효과 | 클리어/DPS/서포트 | **물리/마법/지원 3속성** |
| 히든 유닛 | 없음 | 제한적 | **조합표 기반 히든 유닛** |

### 1.4 엘리베이터 피치

"Random Dice의 빠른 PvP와 직관적인 모바일 UX에, SC 랜덤 디펜스의 깊은 합성 시스템과 유닛 조합 전략을 결합한 게임. 5x4 확장 보드에서 20개 슬롯을 활용한 배치 전략, 합성 시 타입 유지/변경 선택권, 히든 유닛 조합 시스템이 핵심 차별점이다."

## 2. 핵심 게임플레이 (Core Gameplay)

### 2.1 보드 시스템

#### 그리드 레이아웃: 5x4 (20슬롯)

```
    Col1   Col2   Col3   Col4   Col5
   +------+------+------+------+------+
R1 | [01] | [02] | [03] | [04] | [05] |   ← 후열 (원거리 유닛 선호)
   +------+------+------+------+------+
R2 | [06] | [07] | [08] | [09] | [10] |   ← 중열
   +------+------+------+------+------+
R3 | [11] | [12] | [13] | [14] | [15] |   ← 중열
   +------+------+------+------+------+
R4 | [16] | [17] | [18] | [19] | [20] |   ← 전열 (근접/탱크 유닛 선호)
   +------+------+------+------+------+
         ↑                        ↑
       경로 시작              경로 종료
```

#### 적 이동 경로 (S자 형태)

```
    START →→→→→→→→→→→→→→→→→→→→→→→
                                  ↓
    ←←←←←←←←←←←←←←←←←←←←←←←←←←←
    ↓
    →→→→→→→→→→→→→→→→→→→→→→→→ END → (기지 도착 = 데미지)
```

- 경로는 보드 외곽을 따라 S자로 이동
- 유닛은 슬롯에 고정, 사거리 내 적을 자동 공격
- 기본 사거리: 인접 2칸 (유닛별 상이)

#### 슬롯 특성
- **전열 보너스:** Row 4에 배치된 근접 유닛은 공격력 +10%
- **후열 보너스:** Row 1에 배치된 원거리 유닛은 사거리 +1
- **중앙 보너스:** Col3에 배치된 지원 유닛은 버프 범위 +1

### 2.2 유닛 소환

#### 덱 구성
- 전투 전 **5개 유닛**으로 덱 구성 (보유 유닛 중 선택)
- 덱 내 유닛만 전투 중 소환 가능
- 각 유닛의 소환 확률은 등급에 따라 결정

#### 소환 확률 (기본)

| 등급 | 확률 | 비고 |
|------|------|------|
| 노말 (Normal) | 60% | 기본 유닛, 합성 재료 |
| 레어 (Rare) | 25% | 주력 딜러 |
| 에픽 (Epic) | 12% | 핵심 전력 |
| 전설 (Legendary) | 3% | 게임 체인저 |

- 소환 시 덱 내 해당 등급 유닛 중 **균등 확률**로 선택
- 예: 덱에 노말 2개 → 노말 확정 시 각 50%

#### 소환 비용 (SP)

| 소환 횟수 | 비용 | 누적 비용 |
|-----------|------|-----------|
| 1~5회 | 50 SP | 250 SP |
| 6~10회 | 70 SP | 600 SP |
| 11~15회 | 100 SP | 1,100 SP |
| 16~20회 | 150 SP | 1,850 SP |

**공식:** `cost = base_cost * (1 + floor(summon_count / 5) * 0.4)`
- `base_cost = 50 SP`
- 5회마다 단계 상승
- 최대 비용 캡: 200 SP

### 2.3 합성 시스템

합성은 JayGame의 **핵심 전략 요소**이다.

#### 합성 조건
- **같은 타입** + **같은 레벨(성)** = 합성 가능
- 드래그 앤 드롭으로 합성 실행

#### 합성 결과: 선택 시스템

합성 시 플레이어에게 **2가지 선택지** 제공:

```
┌─────────────────────────────────┐
│     합성 결과를 선택하세요!      │
│                                 │
│  ┌───────────┐  ┌───────────┐  │
│  │  [유지]    │  │  [변환]    │  │
│  │           │  │           │  │
│  │  화염 ★3  │  │  ???  ★3  │  │
│  │  (확정)   │  │  (랜덤)   │  │
│  │           │  │           │  │
│  └───────────┘  └───────────┘  │
│                                 │
│   선택 제한시간: 5초            │
│   ████████████░░ 3.2초         │
└─────────────────────────────────┘
```

| 선택지 | 설명 | 전략적 의미 |
|--------|------|-------------|
| **유지** | 같은 타입, 레벨+1 | 안정적. 덱 빌드 유지 |
| **변환** | 랜덤 타입, 레벨+1 | 도박성. 더 높은 등급 유닛 획득 가능 |

#### 변환 시 등급 확률

| 결과 등급 | 확률 |
|-----------|------|
| 같은 등급 | 70% |
| 한 단계 높은 등급 | 20% |
| 한 단계 낮은 등급 | 8% |
| 두 단계 높은 등급 | 2% |

- 노말 유닛 변환 시: 노말 70%, 레어 20%, 에픽 8%, 전설 2%
- 전설 유닛 변환 시: 전설 90%, 에픽 10% (하락만 가능)
- **선택 제한시간:** 5초 (미선택 시 자동 "유지")

#### 히든 유닛 조합

특정 유닛 조합을 같은 보드에 배치하면 **히든 유닛** 해금 가능:

```
[화염 ★5] + [냉기 ★5] → [폭풍 ★5] (히든)
조건: 두 유닛이 인접 슬롯에 배치되어야 함
```

| 조합 | 결과 (히든 유닛) | 효과 |
|------|------------------|------|
| 화염 ★5 + 냉기 ★5 | 폭풍 | 범위 공격 + 슬로우 |
| 독 ★5 + 치유 ★5 | 역병 | 적 회복 불가 + 광역 DoT |
| 번개 ★5 + 철벽 ★5 | 자기장 | 주변 아군 실드 + 체인 공격 |
| 저격 ★5 + 강화 ★5 | 레일건 | 관통 + 극대화 크리티컬 |

- 히든 유닛 조합표는 게임 내에서 점진적으로 해금
- 조합 성공 시 두 유닛이 합쳐져 1슬롯 차지 (슬롯 효율 증가)

### 2.4 SP 경제 시스템

#### SP 획득 방법

| 획득원 | SP량 | 비고 |
|--------|------|------|
| 일반 적 처치 | 5 SP | 기본 수입 |
| 빠른 적 처치 | 8 SP | 이동속도 빠름 |
| 탱크 적 처치 | 15 SP | HP 높음 |
| 비행 적 처치 | 12 SP | 대공 유닛 필요 |
| 보스 처치 | 50 + (wave * 5) SP | 웨이브 비례 |
| 웨이브 클리어 보너스 | 20 SP | 웨이브 완료 시 |
| 시간 경과 보너스 | 2 SP/초 | 기본 수입 |
| 유닛 매각 | 소환비용 * 0.5 | 슬롯 관리용 |

#### SP 소비처

| 소비처 | 비용 | 비고 |
|--------|------|------|
| 유닛 소환 | 50~200 SP | 점진적 증가 |
| 유닛 매각 후 재배치 | 10 SP | 위치 변경 |
| 스킬 발동 | 100~300 SP | 덱 스킬 |
| 긴급 수리 (HP 회복) | 200 SP | PvE 전용 |

#### 경제 밸런스 공식
- 웨이브당 평균 적 수: `10 + wave * 2`
- 웨이브당 예상 SP 수입: `(적수 * 평균처치SP) + 클리어보너스 + 시간보너스`
- 웨이브 10 기준: `(30 * 7) + 20 + (30 * 2) = 290 SP`
- 소환 가능 횟수 = `총 SP / 현재 소환 비용`
- **목표:** 플레이어가 매 웨이브 1~2회 소환 가능한 경제 구조

### 2.5 전투 시스템

#### 유닛 자동 공격
- 유닛은 사거리 내 적을 **자동으로 공격**
- 공격 대상 우선순위: 가장 앞에 있는 적 (경로 기준)
- 복수 대상 공격 유닛은 별도 표기

#### 공격력 계산
```
최종 공격력 = base_atk * level_multiplier * buff_multiplier * position_bonus

level_multiplier = 1.0 + (level - 1) * 0.5
  - ★1 = 1.0x
  - ★2 = 1.5x
  - ★3 = 2.0x
  - ★4 = 2.5x
  - ★5 = 3.0x
  - ★6 = 3.5x
  - ★7 = 4.0x

buff_multiplier = 1.0 + sum(active_buffs)
position_bonus = 1.0 + slot_bonus (0.0 ~ 0.1)
```

#### 데미지 타입

| 타입 | 설명 | 유리한 적 | 불리한 적 |
|------|------|-----------|-----------|
| 물리 | 기본 데미지 | 일반 | 탱크 (50% 감소) |
| 마법 | 방어 관통 | 탱크 | 비행 (30% 감소) |
| 순수 | 감소 없음 | 모든 적 | 없음 (희귀) |

## 3. 유닛 시스템 (Unit System)

### 3.1 유닛 등급

| 등급 | 색상 | 기본 공격력 범위 | 기본 공격속도 | 유닛 수 |
|------|------|-----------------|--------------|---------|
| 노말 (Normal) | 회색/흰색 | 10~20 | 1.0초 | 10개 |
| 레어 (Rare) | 파랑 | 20~35 | 0.8~1.2초 | 12개 |
| 에픽 (Epic) | 보라 | 35~60 | 0.6~1.5초 | 10개 |
| 전설 (Legendary) | 금색 | 50~100 | 0.5~2.0초 | 8개 |

**총 유닛 수:** 40개 (출시 기준)

### 3.2 유닛 속성 (3속성 시스템)

| 속성 | 아이콘 | 역할 | 시너지 |
|------|--------|------|--------|
| **물리 (Physical)** | 검 | 직접 데미지 딜러 | 물리 3개 이상: 공격력 +15% |
| **마법 (Magic)** | 지팡이 | 범위/특수 효과 | 마법 3개 이상: 스킬 쿨다운 -20% |
| **지원 (Support)** | 방패 | 버프/디버프 | 지원 2개 이상: 버프 효과 +25% |

#### 속성 시너지 보너스 (보드 위 유닛 기준)

```
보드 위 같은 속성 유닛 수 → 시너지 발동

물리 3/5/7개: 공격력 +15% / +25% / +40%
마법 3/5/7개: 쿨다운 -20% / -30% / 스킬 데미지 +50%
지원 2/4/6개: 버프 효과 +25% / +40% / 전체 유닛 공속 +20%
```

#### 적 HP 스케일링 공식

```
enemy_hp = base_hp * difficulty_multiplier * (1.08 ^ wave_number)

base_hp 기준:
  일반: 100
  빠른: 60
  탱크: 300
  비행: 80
  보스: base_hp * boss_multiplier (10x ~ 100x)

예시 (Normal 난이도):
  웨이브 1 일반: 100 * 1.0 * 1.08^1 = 108
  웨이브 10 일반: 100 * 1.0 * 1.08^10 = 216
  웨이브 20 일반: 100 * 1.0 * 1.08^20 = 466
  웨이브 30 일반: 100 * 1.0 * 1.08^30 = 1,006
  웨이브 40 일반: 100 * 1.0 * 1.08^40 = 2,172
```

#### 적 등장 수 공식

```
enemy_count = base_count + (wave * 2)

base_count = 8
웨이브 1: 10마리
웨이브 10: 28마리
웨이브 20: 48마리
웨이브 30: 68마리 (데스웨이브: 속도 증가)
```

### 8.4 게임 루프 구조

```cpp
// 메인 게임 루프 (60 FPS 타겟)
while (running) {
    // 1. Input Processing (< 1ms)
    processInput();

    // 2. Game Logic Update (< 5ms)
    float dt = swappy_getDeltaTime();  // ~16.67ms
    updateGameState(dt);
    updateUnits(dt);
    updateEnemies(dt);
    updateProjectiles(dt);
    checkCollisions();
    updateUI(dt);

    // 3. Render (< 8ms)
    beginFrame();
    renderBackground();
    renderPath();
    renderEnemies();
    renderUnits();
    renderProjectiles();
    renderParticles();
    renderUI();
    endFrame();

    // 4. Frame Pacing (Swappy)
    SwappyGL_swap(display, surface);
}
```

## 부록 A: 밸런스 시뮬레이션 기준값

### 웨이브별 DPS 요구량 (PvE Normal)

```
필요 DPS = (적수 * 적HP) / 웨이브 시간(초)

웨이브 1:  (10 * 108) / 90  =    12 DPS
웨이브 5:  (18 * 147) / 90  =    29 DPS (+ 보스별도)
웨이브 10: (28 * 216) / 90  =    67 DPS
웨이브 15: (38 * 317) / 90  =   134 DPS
웨이브 20: (48 * 466) / 90  =   249 DPS
웨이브 25: (58 * 685) / 85  =   467 DPS
웨이브 30: (68 * 1006) / 80 =   855 DPS
웨이브 35: (78 * 1478) / 70 = 1,647 DPS
웨이브 40: (88 * 2172) / 60 = 3,186 DPS
```

### 예상 보드 DPS (이상적 플레이)

```
웨이브 10 기준 (보드 ~8유닛 평균 ★2):
  8 * 20(avg ATK) * 1.5(★2) / 1.0(avg 공속) = 240 DPS ✓ (여유)

웨이브 20 기준 (보드 ~14유닛 평균 ★3):
  14 * 25(avg ATK) * 2.0(★3) / 0.9(avg 공속) = 778 DPS ✓ (적절)

웨이브 30 기준 (보드 ~18유닛 평균 ★4):
  18 * 30(avg ATK) * 2.5(★4) / 0.85 = 1,588 DPS ✓ (타이트)

웨이브 40 기준 (보드 ~20유닛 평균 ★5, 히든 포함):
  20 * 40(avg ATK) * 3.0(★5) / 0.8 = 3,000 DPS ~ (극한)
```

### 밸런스 안전장치
- 웨이브 40 클리어율 목표: **10~15%** (Normal 난이도)
- DPS 부족 구간: 유닛 특수 능력(DoT, 디버프)으로 보정
- 보스 DPS 체크 실패 시: 보스가 탈출하여 기지 데미지 (강제 진행)
