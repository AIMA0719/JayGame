# JayGame UI 전면 리디자인 - 다크 네온 스타일

## 개요

메이플 랜덤 디펜스 모바일 게임의 UI를 운빨존많겜/S2RD 레퍼런스 수준으로 전면 리디자인한다.

**확정 사항:**
- 접근: 하이브리드 (배틀 필드=C++ OpenGL, 모든 UI/HUD=Compose)
- 스타일: 다크 네온 (운빨존많겜 참고)
- 배틀 HUD: 상단+하단 전부 Compose 오버레이
- 메뉴 화면: 8개 전부 리디자인
- C++ 렌더링: GL_LINEAR + 스프라이트 해상도 개선
- 데이터: 목데이터 금지, 실제 데이터 연동
- 테스트 포함

- 상단: 현재 덱 5슬롯 (가로 배치, GameCard)

- 패배: NeonRed 타이틀
- 스탯 카드: 웨이브, 킬, 머지, 획득 골드/트로피
- 홈으로 버튼

### 2.6 ShopScreen
- 탭: 골드팩 / 다이아팩 / 특별
- 상품 카드: 가격 + 내용물 미리보기

### 2.7 SeasonPassScreen

### 2.8 AchievementsScreen

### 2.9 SettingsScreen
- 토글 스위치 (사운드, 음악)

- 데이터 초기화 (확인 다이얼로그)

## 3. C++ 렌더링 개선

### 3.1 텍스처 필터링
- `TextureAsset.cpp`: `GL_NEAREST` → `GL_LINEAR` (line 52-53, 87-88)
- 배틀 필드 스프라이트가 부드럽게 보이도록

### 3.2 스프라이트 해상도
- SpriteAtlas 셀 크기: 64x64 → 128x128 유지 가능 (GL_LINEAR로 충분)
- 아틀라스 크기 2048x2048 유지

### 3.3 C++ UISystem/TextRenderer
- 배틀 HUD를 Compose로 옮기므로, C++ UISystem의 HUD 렌더링 코드 비활성화
- TextRenderer는 디버그 용도로만 유지

## 4. C++↔Compose 통신 (JNI 브릿지)

### 필요한 실시간 데이터 (C++ → Compose)
- 현재 웨이브 / 최대 웨이브
- 적 HP (보스일 때)
- SP 잔량
- 경과 시간
- 배틀 상태 (Playing, Victory, Defeat)

### Compose → C++ 콜
- 소환 버튼 클릭 → summonUnit()
- 일시정지 → pauseGame()

### 구현 방식
- `GameEngine`에 콜백 등록 함수 추가
- JNI를 통해 Kotlin 함수를 주기적으로 호출 (매 프레임 X, 상태 변경 시에만)
- StateFlow로 Compose UI 자동 갱신

## 5. 데이터 연동

- 모든 화면에서 GameRepository.gameData StateFlow 사용
- 목데이터 절대 금지
- 배틀 결과 → SaveSystem(C++) → SaveBridge(JNI) → GameRepository → Compose UI
- 스태미나, 골드, 다이아 등 실시간 반영

## 6. 테스트 전략

- 각 화면 네비게이션 동작 확인
- 배틀 HUD 데이터 갱신 확인 (웨이브, SP, HP)
- 소환 버튼 클릭 → 실제 유닛 소환 확인
- 덱 편집 → 저장 → 배틀 반영 확인
- 골드/다이아 소비 → 잔액 반영 확인
- 스태미나 소비 → 리젠 확인
- 화면 전환 시 데이터 유지 확인
